Evaluated feature parity between Claude Code and VS Code Copilot

Great question. The short answer is: *not directly* — Claude Code's |/skills| and |/agent| are specific to the Claude Code CLI runtime. But GitHub Copilot in VS
Code has its own equivalents you can leverage, especially if Anthropic
models are available as the backend.

Here's how the mapping works:

*Claude Code |/skills| → Copilot Custom Instructions + Prompt Files*

VS Code Copilot supports two mechanisms that approximate skills:

  * *|.github/copilot-instructions.md|* — a repo-level file that's automatically injected into every
    Copilot chat in that workspace. You can put your project context,
    coding conventions, or tool-specific instructions here (e.g., "this
    project uses Rocky Linux, MySQL, and Flask").
  * *Reusable |.prompt.md| files* — stored in |.github/prompts/|, these are named prompt templates you can invoke from the Copilot
    Chat panel with |#| references. Think of them as manually-triggered skills.

*Claude Code |/agent| → Copilot Agent Mode*

Copilot's *Agent Mode* (available in VS Code Insiders and rolling out to
stable) lets Copilot autonomously plan multi-step tasks, run terminal
commands, edit multiple files, and iterate — very similar to Claude
Code's agent behaviour. You invoke it by switching the chat mode from
"Ask" to "Agent" in the chat panel. With Anthropic models selected as
the backend, it'll use Sonnet/Opus for the reasoning.

*MCP Servers (the more powerful bridge)*

If your company's Copilot setup supports MCP (Model Context Protocol),
you can configure MCP servers in VS Code's |settings.json| under |github.copilot.chat.mcpServers|. This gives agent mode access to external tools — file systems, APIs,
databases — which is closer to Claude Code's full agent capability.

*Practical limitations to be aware of:*

The Anthropic model selection in Copilot is routed /through GitHub's
infrastructure/, not directly through the Anthropic API. So you won't
have the same fine-grained control over system prompts or tool
definitions that Claude Code gives you natively. The |/skills| concept in Claude Code is also more structured (with defined tool
schemas), whereas Copilot's equivalent is more freeform prompt injection.

*Bottom line for your setup:* Start with |.github/copilot-instructions.md| for persistent context (great for your IT/ITSM stack context — MySQL
schemas, Rocky Linux environment, etc.) and explore switching to Agent
Mode for multi-step coding tasks. That combination gets you closest to
the Claude Code experience without needing to install Claude Code itself.

Good question — and yes, you can, with a bit of manual adaptation.
Claude Code skills are stored as files on your local machine, so you can
read them and reformat them for Copilot.


    Step 1: Find Your Skill File

On Windows, Claude Code skills are typically stored here:

|C:\Users\YourUsername\.claude\skills\tech-web-designer\|

Open that folder and you'll find a |SKILL.md| (or similar) file containing the skill's instructions, context, and rules.


    Step 2: Adapt It to |.prompt.md| Format

Claude Code skills and Copilot prompt files are both just markdown — but
the frontmatter and variable syntax differ slightly. Here's the mapping:

Claude Code Skill	GitHub Copilot |.prompt.md|
Internal tool schema/calls	Describe the intent in plain text
|$ARGUMENTS|	|${input}|
No frontmatter needed	Add |mode:| and |description:| frontmatter
Tool use blocks	Ask Copilot to take the equivalent action


    Step 3: Example Conversion

Say your |tech-web-designer| skill in Claude Code looks something like this:

markdown

|You are a skilled web designer and frontend developer.
Create modern, responsive web pages using HTML, CSS, and JavaScript.
Follow these conventions:
- Use CSS variables for theming
- Mobile-first responsive design
- Semantic HTML5 elements
- Vanilla JS unless a framework is specified|

You'd convert it to |.github/prompts/tech-web-designer.prompt.md| like this:

markdown

|---
mode: agent
description: Design and build a modern, responsive web page
---

You are a skilled web designer and frontend developer.
Create modern, responsive web pages using HTML, CSS, and JavaScript.

## Design conventions:
- Use CSS variables for theming (define all colours/fonts in :root)
- Mobile-first responsive design using flexbox or CSS grid
- Semantic HTML5 elements (nav, main, section, article, etc.)
- Vanilla JS unless a framework is explicitly requested
- Clean, well-commented code

## Deliverables:
- Produce complete, ready-to-run HTML/CSS/JS files
- If multiple files are needed, create them all
- Briefly explain design decisions made

Task: ${input}|


    The One Limitation

The main thing you *lose* in translation is any *tool calls or file
system actions* baked into the Claude Code skill. Claude Code skills can
directly invoke tools (read files, run bash, etc.) as part of their
definition. Copilot prompt files can't define tools — they can only /
instruct/ the agent to take actions, which it then decides how to execute.

So the skill logic itself translates fine — it's just the low-level tool
wiring that doesn't carry over directly.
